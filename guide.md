# 使い方ガイド

雀魂のAI評価システム「MAKA」と対局順位を記録・分析するアプリです。
対局を積み重ねることで、自分の打ち筋と実力の傾向が見えてきます。

---

## 基本的な使い方

### 1. 対局を記録する（入力タブ）

対局が終わったら「入力」タブから記録します。

**必須項目**（すべて入力が必要です）

| 項目 | 説明 |
|------|------|
| 日付・時刻 | 対局した日時。同日に複数対局しても問題ありません |
| 戦型 | 東風戦 / 半荘戦 |
| 対局ルーム | 銅の間〜王座の間 |
| 順位 | 1〜4位 |
| MAKA評価 | 牌譜分析後にMAKAが表示する評価（S+〜E） |

**任意項目**（入力するとより詳細な分析が可能になります）

| 項目 | 説明 |
|------|------|
| 最終持ち点 | 対局終了時の素点（段位ポイントではありません） |
| 和了回数 | その対局で和了した回数 |
| 放銃回数 | その対局で放銃した回数 |
| 牌譜URL | 雀魂の牌譜共有URL |
| メモ | 気づいたことや反省点など自由に |

> **牌譜URLについて**  
> 雀魂の共有機能でコピーされる「雀魂牌譜: https://...」形式のテキストをそのまま貼り付けてもURLだけ自動で抽出されます。

---

## MAKA評価について

MAKAは雀魂に搭載されたAI評価システムです。牌譜を分析し、その対局の打ち手の質をS+〜Eで評価します。

| 評価 | 数値 | 目安 |
|------|------|------|
| S+ | 0 | 最高評価 |
| S | 1 | |
| S- | 2 | |
| A+ | 3 | |
| A | 4 | 平均的な上級者 |
| A- | 5 | |
| B+ | 6 | |
| B | 7 | |
| B- | 8 | |
| C+ | 9 | |
| C | 10 | |
| C- | 11 | |
| D+ | 12 | |
| D | 13 | |
| D- | 14 | |
| E | 15 | ほぼ出現しない |

数値が小さいほど高評価です。統計の計算にはこの数値が使われ、グラフや表では元の記号（S+など）に変換して表示されます。

## 統計分析（統計タブ）

記録が溜まると統計タブで様々な分析が見られます。

## 大切な前提

麻雀は（麻雀に限らずですが）、実力が統計情報に収束するのには**膨大な試合数**が必要です。  
また、このツールを使ってみたいほとんどの方は「上手になりたい」と思いながら麻雀に取り組むでしょう。その前提では、多かれ少なかれ実力は下手→上手の方向で推移するため、そもそも仮に100万局打ったとして「実力に収束」という表現も完全に適切ではないかもしれません。  
あくまで当ツールでの統計情報は**傾向**を見るものとしてご使用ください。

### フィルター

統計タブの上部で集計対象を絞り込めます。

- **戦型フィルター**：すべて / 東風戦 / 半荘戦
- **ルームフィルター**：すべて / 銅〜王座

2つを組み合わせることができます。例えば「半荘戦の玉の間だけ」といった絞り込みが可能です。

### サマリーカード

- **総対局数**：フィルター条件に合致する対局数
- **平均順位**：数値が小さいほど好成績
- **平均MAKA**：数値が小さいほど高評価
- **1位率**：1位を取った割合

### MAKA評価分布（帯グラフ）

全対局のMAKA評価の内訳を100%積み上げ横棒グラフで表示します。自分のMAKA評価がどの帯に集中しているかが一目でわかります。

### 順位別 MAKA評価分布

1〜4位それぞれの対局で、MAKA評価がどう分布しているかを帯グラフで表示します。「1位を取った対局でも実はMAKAは低かった」「4位でもMAKAが高い（運が悪かっただけ）」といった傾向が読み取れます。

### MAKA評価ごとの順位統計

MAKAがS+の対局では平均何位か、などを集計します。平均・中央値・最頻値の3種類を表示します。最頻値は同率の場合すべて表示します。

### 順位ごとの MAKA評価統計

1位の対局では平均MAKA評価がどれくらいか、などを集計します。

### ルームランク別統計

ルームごとの平均MAKA・平均順位・最頻値を集計します。フィルターとは独立して**全記録**を対象に集計します。

### MAKA評価ごとの平均データ（任意入力あり）

持ち点・和了回数・放銃回数を入力した対局のみを対象に、MAKA評価ごとの平均値を表示します。任意項目を入力していない場合はこのセクションは表示されません。

### 時間帯別 MAKA評価分布

時間帯ごとのMAKA評価分布を帯グラフと統計テーブルで表示します。時間帯の区切りは「設定タブ」で自由に変更できます。「深夜帯はMAKAが下がる」といった傾向の発見に使えます。

### 連戦 N局目別 MAKA評価分布

同じ日に連続して対局した場合、何局目かでMAKA評価がどう変わるかを分析します。**前の対局から2時間以内**を「連戦」と判定します。4局目以降はまとめて集計されます。「連戦3局目から集中力が落ちる」といった傾向を確認できます。

---

## 対局ログ（ログタブ）

記録した対局を一覧で確認できます。新しい順に並んでいます。

- **行をクリック**すると詳細モーダルが開き、すべての入力内容を確認できます
- モーダル内の「編集」ボタンで内容を修正できます
- 牌譜URLを登録した対局は「牌譜」リンクから直接開けます
- 各行右端の「削除」ボタン、またはモーダル内の「この記録を削除」から削除できます

---

## 設定（設定タブ）

### 時間帯の設定

時間帯分析に使う時間帯の区切りを自由にカスタマイズできます。

- **帯名**と**開始時刻**を組み合わせて設定します
- 「＋ 帯を追加」でいくつでも追加できます
- 「設定を保存」を押すと開始時刻順に自動ソートされて保存されます
- 設定変更後は統計タブを開き直すと反映されます

**デフォルト設定**

| 帯名 | 開始時刻 |
|------|----------|
| 朝 | 06:00 |
| 昼 | 12:00 |
| 夜 | 18:00 |
| 深夜 | 00:00 |

## データ管理（データタブ）

### エクスポート

「JSONエクスポート」ボタンで全記録をJSONファイルとしてダウンロードできます。定期的にバックアップしておくことをおすすめします。

### インポート

「JSONインポート」ボタンでJSONファイルを読み込めます。読み込み時に以下を選択できます。

- **追記**：既存データに追加します。IDが重複するレコードはスキップされます
- **上書き**：既存データをすべて削除して置き換えます

### 全データ削除

「全データ削除」ボタンで全記録を削除できます。この操作は元に戻せません。事前にエクスポートしてバックアップを取ることをおすすめします。

---

## データの保存について

このアプリはすべてのデータをブラウザの**localStorage**に保存しています。サーバーには一切送信されません。

- ブラウザのデータを消去すると記録も消えます。定期的にエクスポートでバックアップしてください
- 別のブラウザや端末とは自動で同期されません。引き継ぎにはエクスポート→インポートを使ってください
- シークレット（プライベート）ウィンドウでは、ブラウザを閉じるとデータが消えます

## 統計用語の解説

このアプリの統計タブでは、いくつかの統計指標を使って成績を分析しています。数学的な背景がなくても読めるよう、麻雀の文脈で説明します。

---

### 平均（average）

すべての値を足して、件数で割ったものです。

> **例）** A評価で打った10局の順位が 1, 2, 1, 3, 2, 1, 4, 2, 1, 3 だった場合、合計は20なので平均順位は **2.0**。

最もよく使われる指標ですが、極端な値に引っ張られやすい弱点があります。

---

### 中央値（median）

全データを順番に並べたとき、ちょうど真ん中に来る値です。極端な値に引っ張られない点が平均と異なります。

> **例）** MAKA数値（S+=0〜E=15）が 2, 3, 3, **4**, 5, 6, 8 なら中央値は **4（= A）**。

このアプリでは **MAKA評価の列にのみ中央値を表示しています**。順位（1〜4の4種類）には表示していません。理由は、順位のような取りうる値が少ない指標では、データが増えると中央値が必ず 2・2.5・3 のどれかに収束してしまい、ほとんど情報が読み取れなくなるためです。一方MAKAは16段階あるため、中央値にも一定の意味があります。

---

### 最頻値（mode）

最も多く出現した値のことです。

> **例）** 10局の順位が 1, 1, 1, 2, 2, 3, 3, 3, 4, 4 なら、1位と3位がどちらも3回で同率1位なので最頻値は **1・3**。

「このMAKA評価のとき、いちばん多く出た順位」を知りたいときに使います。最頻値が複数ある場合はすべて表示されます。

---

### 標準偏差（standard deviation）

値がどれだけ平均から散らばっているかを表す指標です。値が大きいほど「ばらつきが大きい（結果が安定しない）」ことを意味します。

> **例）** A評価の順位の標準偏差が **0.8** → 毎回1〜2位に収まることが多く安定している。  
> **例）** A評価の順位の標準偏差が **1.5** → 1位になることも4位になることもあり、結果が読めない。

標準偏差が小さい = MAKAと順位の関係が安定している（実力が出やすい）。  
標準偏差が大きい = 同じMAKAでも順位がバラバラ（運の影響が大きい）。

---

### 四分位数・Q1〜Q3（quartile）

データを小さい順に並べたとき、**下から25%地点の値がQ1（第1四分位数）**、**75%地点の値がQ3（第3四分位数）** です。Q1〜Q3の範囲に全体の中央50%のデータが収まります。

> **例）** S評価12局の順位のQ1が1、Q3が2なら、「S評価の局の半数は1〜2位の間に収まっている」ことを意味します。

平均や中央値の一点だけでなく、「典型的な結果の幅」がわかります。Q1〜Q3の幅が狭いほど安定、広いほどばらつきが大きいと読めます。

---

### 最小〜最大（min / max）

記録された中で最も良かった値と最も悪かった値です。

> **例）** S+評価の順位の最小〜最大が **1〜4** → S+を取っても4位になったことがある。  
> **例）** C評価の最小〜最大が **1〜4** → C評価でも1位を取ったことがある。

平均では見えない「極端なケース」を確認するための指標です。「どんなに良い評価でも運次第で負けることがある」「低い評価でも運良く勝てることがある」という麻雀の性質をそのまま数値で確認できます。

---

### 相関係数（correlation coefficient）

2つの指標がどれくらい連動しているかを **-1〜+1** の数値で表します。このアプリではMAKA数値（小=高評価）と順位（小=好成績）の間の**スピアマン順位相関係数**を表示しています。

> **例）** 相関係数 **+0.55** → MAKAが良いほど順位も良い傾向がある（中程度の正の相関）。  
> **例）** 相関係数 **+0.08** → MAKAと順位はほぼ無関係（運の影響が非常に大きい）。

MAKA数値と順位はどちらも「小さいほど良い」ので、両者が連動していれば相関係数はプラスに近づきます。

**ピアソンではなくスピアマンを使う理由**  
ピアソン相関係数は「値の間隔が等しい」ことを前提とします。しかしMAKA評価のS+とSの実力差が、CとC-の実力差と本当に同じかどうかは不明です。スピアマン相関係数は値を順位に変換してから計算するため、この前提が不要で、MAKA評価のような指標に対してより適切です。

| 相関係数 | 解釈 |
|----------|------|
| 0.7以上 | 強い相関。実力が結果に出やすい状態 |
| 0.4〜0.7 | ある程度の相関。概ね実力が反映されている |
| 0.2〜0.4 | 弱い相関。運の影響も大きい |
| 0.2未満 | ほぼ無相関。運の影響が非常に大きい |

相関係数は5局以上のデータがある場合に表示されます。

---

### 指標をまとめて読む

統計の各指標は単独で読むより、組み合わせることで意味が深まります。

| 状況 | 読み取れること |
|------|--------------|
| 平均MAKAと中央値MAKAが大きく離れている | ごく稀に飛び抜けて良い／悪い評価の局がある |
| 標準偏差が小さい＋Q1〜Q3が狭い | そのMAKA評価帯・順位では結果が安定している |
| 標準偏差が大きい＋最小〜最大が1〜4 | 運の影響が大きく、MAKAだけでは順位が決まっていない |
| Q1=1・Q3=4（順位） | 1位と4位が半々の「博打型」。平均2.5でも安定型とは全く異なる |

データが少ない段階では指標がぶれやすいため、**最低20〜30局**を目安に解釈することをおすすめします。
